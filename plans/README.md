## Direction Options

- Finish the Rust audio pilot only after a benchmark proves it beats Swift on the live metering path. The repo already has Phase 0/2 reports and staging code, so the next move should be measurement and removal of obvious overhead, not a broad rewrite.
- Make imported audio a deliberate product tier. Either keep it as raw transcription only and make that clear, or allow "meeting import" so past calls can use the same summary, Q&A, and notes workflow as live captures.
- Treat storage/query scalability as the next foundation item before expanding history-heavy analytics. Current open issues already point to FTS/FRC/storage performance; plan 001 is a narrow first slice that does not require a full UI architecture migration.
